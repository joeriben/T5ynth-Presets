# T5ynth-Presets — *UCDCAE AI Lab* read-only preset bank

This repository holds the **UCDCAE AI Lab** preset collection for
[T5ynth](https://github.com/joeriben/T5ynth) — a JUCE/C++ synthesizer that
treats Stable Audio Open's latent space as a meta-oscillator. Each `.t5p`
file is one complete instrument state: prompts, axes, dimension offsets,
engine parameters, generated audio, and (optionally) an inference cache.

## For T5ynth users — how to get these presets

You do **not** need to clone this repo. Open T5ynth, go to the preset
panel, and press **Update Library** under the *UCDCAE AI Lab* bank. The
plugin reads `manifest.json` from this repo, downloads any new or changed
`.t5p` files via `raw.githubusercontent.com`, and deletes local files the
maintainer has removed upstream. The bank lives in the user-writable
preset directory, so updates never need elevated privileges.

The *UCDCAE AI Lab* bank is **read-only inside the plugin**: rename,
duplicate, and tag-edit are disabled to keep the local copy in sync with
upstream. Save your own variations into the User bank instead. You may
still delete an upstream preset locally if you do not like it — the next
update will simply re-pull it.

## Layout

```
T5ynth-Presets/
├── manifest.json                  ← generated by CI (do not hand-edit)
├── Analog Saw C2C3.t5p            ← every .t5p lives at the repo root
├── Heaven and Hell WT.t5p
├── ...
├── scripts/
│   └── generate_manifest.py       ← rebuilds manifest.json
└── .github/workflows/manifest.yml ← triggers on every push that touches a .t5p
```

## For the maintainer — adding or removing a preset

```bash
# add a preset
cp ~/Desktop/MyNewPreset.t5p .
git add MyNewPreset.t5p
git commit -m "add: MyNewPreset"
git push
# ~30 s later: manifest.json is regenerated by GitHub Actions
# next user click on "Update Library" pulls the new preset

# remove a preset
git rm "Some Old Thing.t5p"
git commit -m "remove: Some Old Thing"
git push
```

The path filter in `manifest.yml` matches only `*.t5p` at the root, so the
workflow's own commit to `manifest.json` does **not** re-trigger itself —
no `[skip ci]` needed.

## Manifest schema (v1)

```json
{
  "schema_version": 1,
  "generated_at": "2026-05-17T13:45:16Z",
  "base_url": "https://raw.githubusercontent.com/joeriben/T5ynth-Presets/main/",
  "presets": [
    {
      "name": "Heaven and Hell WT",
      "path": "Heaven and Hell WT.t5p",
      "size": 2082559,
      "sha256": "..."
    }
  ]
}
```

The plugin keys deduplication on `path` and refresh decisions on `sha256`,
so silent corruption of a local file or an upstream content change with
identical byte count both trigger a re-download.

## Manual generation (no CI)

```bash
python scripts/generate_manifest.py
git add manifest.json
git commit -m "regenerate manifest"
git push
```
